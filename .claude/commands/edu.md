# /edu

학습 목적의 기술 문서를 작성하여 Confluence에 게시한다.

## Arguments
- $ARGUMENTS: 학습 주제
    - 주제 (필수): 학습할 기술 주제
    - 난이도 (선택): 입문, 중급, 고급 (기본: 중급)

## Instructions

### Step 1: 주제 분석

1. 입력된 주제를 분석한다
2. 대상 독자 수준을 파악한다 (입문/중급/고급)
3. 문서 유형을 결정한다:
    - **Concept**: 개념 설명
    - **Tutorial**: 단계별 실습
    - **Reference**: 레퍼런스 가이드
    - **Deep Dive**: 심층 분석
4. 관련 서브 주제를 식별한다 (태그로 활용)

### Step 2: 인덱스(목차) 제안

**중요**: 반드시 문서 작성 전에 인덱스를 먼저 제안하고 사용자 피드백을 받는다.

인덱스 제안 형식:
```
## 제안 인덱스

### 문서 정보
- **제목**: {제목}
- **유형**: {Concept/Tutorial/Reference/Deep Dive}
- **난이도**: {입문/중급/고급}
- **예상 학습 시간**: {시간}

### 목차
1. 개요
   1.1. {하위 주제}
   1.2. {하위 주제}
2. {섹션 제목}
   2.1. {하위 주제}
   2.2. {하위 주제}
3. {섹션 제목}
   ...
4. 결론
5. 관련 키워드
6. 더 깊이 알아보기
7. 면접 질문
8. 근거자료

### 태그 (서브 주제)
- {태그1}: {설명}
- {태그2}: {설명}
- {태그3}: {설명}

이 구조로 진행할까요? 수정이 필요하면 말씀해주세요.
```

### Step 3: 정보 수집

사용자 승인 후 정보를 수집한다.

#### 3.1 기술 문서 수집
1. Context7 MCP 서버를 활용하여 최신 문서를 조회한다
   ```
   resolve-library-id: 라이브러리 ID 조회
   query-docs: 문서 내용 검색
   ```
2. 필요시 codebase를 탐색하여 실제 적용 사례를 확인한다

#### 3.2 근거자료 수집 (필수)

**중요**: 모든 학습자료는 신뢰할 수 있는 근거자료를 기반으로 작성해야 한다.

WebSearch를 활용하여 다음 우선순위로 근거자료를 수집한다:

| 우선순위 | 자료 유형 | 검색 키워드 예시 | 신뢰도 |
|----------|-----------|------------------|--------|
| 0 | 공식문서 | `"{주제}" official documentation` | 최고 |
| 0.5 | RFC 문서 | `"{주제}" site:rfc-editor.org OR site:ietf.org RFC` | 최고 |
| 1 | 논문 | `"{주제}" site:arxiv.org OR site:acm.org OR site:ieee.org` | 최고 |
| 2 | 연구보고서 | `"{주제}" research report site:*.edu OR site:*.gov` | 높음 |
| 3 | 컨퍼런스 | `"{주제}" conference talk KubeCon OR QCon OR Strange Loop` | 높음 |
| 4 | 실험/비교 | `"{주제}" benchmark comparison performance` | 중간 |
| 5 | 기타 | 기술 블로그, 튜토리얼 등 | 낮음 |

**수집 기준**:
- 최소 1개 이상의 논문/연구보고서/컨퍼런스 자료 포함 권장
- 공식 문서는 반드시 포함
- 자료의 발행 연도 확인 (최신 자료 우선)
- 저자/기관의 신뢰도 검증

### Step 4: 학습자료 부모 페이지 확인

Confluence "DOCS" 스페이스에서 "학습자료" 페이지를 확인한다.

```
confluence_search(query='space=DOCS AND title="학습자료"')
```

- 존재하면: 해당 페이지 ID를 parent_id로 사용
- 존재하지 않으면: "학습자료" 페이지를 먼저 생성

### Step 5: 문서 작성

#### 작성 규칙

1. **기술적 정확성**: 공식 문서 기반으로 정확한 정보 제공
2. **코드 예시 필수**: 실행 가능한 예제 코드 포함
3. **시각적 요소 적극 활용** ⭐:
    - **다이어그램**: Mermaid, ASCII 다이어그램으로 구조/흐름 시각화
    - **이모티콘**: 핵심 포인트, 경고, 팁 등에 이모티콘으로 가독성 향상
    - **표**: 비교, 옵션, 속성 등은 반드시 표로 정리
4. **단계별 설명**: 복잡한 개념은 순서대로 분해
5. **근거자료 필수**: 모든 주장/설명에 대한 출처 명시
    - 논문, 연구보고서, 컨퍼런스 자료 우선 인용
    - 근거 없는 주장 금지
    - 자료의 신뢰도 등급 표시

### Step 6: Confluence에 게시

1. 문서를 Confluence에 생성한다
   ```
   confluence_create_page(
     space_key='DOCS',
     title='{제목}',
     content='{Markdown 본문}',
     parent_id='{학습자료 페이지 ID}'
   )
   ```

2. 태그(라벨)를 추가한다
   ```
   confluence_add_label(page_id='{생성된 페이지 ID}', name='{태그}')
   ```
    - 메인 주제 태그
    - 서브 주제 태그들
    - 난이도 태그 (beginner, intermediate, advanced)

## Tag Guidelines

### 메인 태그
- 주 기술/프레임워크명 (예: laravel, redis, docker)

### 서브 태그
- 관련 개념 (예: caching, queue, container)
- 적용 분야 (예: backend, devops, database)
- 패턴/방법론 (예: design-pattern, best-practice)

### 난이도 태그
- `beginner`: 입문
- `intermediate`: 중급
- `advanced`: 고급

### 유형 태그
- `concept`: 개념 설명
- `tutorial`: 실습 가이드

- `deep-dive`: 심층 분석

## Notes

- 인덱스 제안 → 사용자 승인 → 작성 순서 필수

## 근거자료 가이드라인

### 신뢰도 등급

| 등급 | 자료 유형 | 설명 |
|------|-----------|------|
| S | 공식 문서 | 프레임워크/라이브러리 공식 문서 (0순위) |
| S | RFC 문서 | IETF RFC 표준 문서 (0.5순위) |
| A | 논문 (Peer-reviewed) | ACM, IEEE, arXiv 등 학술 논문 |

| B | 실험/비교 자료 | 공신력 있는 벤치마크, 성능 비교 |
| C | 기술 블로그 | 저명한 개발자/기업 기술 블로그 |
| D | 기타 | 일반 블로그, 튜토리얼 등 |
